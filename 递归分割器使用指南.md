# 递归文档分割器使用指南

## 功能特性

递归分割器 `RecursiveDocumentSplitter` 实现了智能的多层级文档分割策略，相比简单的段落分割器，能更好地保留语义完整性。

## 核心优势

1. **递归分割策略**：按优先级尝试不同分隔符，避免强制切断语义
2. **中文优化**：优先按中文标点（。！？）分割，适配中文文档
3. **语义保护**：通过重叠机制减少跨chunk的信息丢失
4. **自适应处理**：自动处理超长段落，确保不超过最大限制

## 分割优先级

```
1. 中文标点：。！？
2. 段落分隔：\n\n
3. 换行符：\n
4. 空格：
5. 字符硬切（最后兜底）
```

## 配置参数

```java
RecursiveDocumentSplitter splitter = new RecursiveDocumentSplitter(
    500,  // maxChunkSize: 最大片段长度
    150   // chunkOverlap: 重叠长度
);
```

## 使用示例

### 基础使用
```java
RecursiveDocumentSplitter splitter = new RecursiveDocumentSplitter(500, 150);
List<TextSegment> segments = splitter.split(document);
```

### 自定义分隔符
```java
List<String> customSeparators = Arrays.asList("\n\n", "\n", " ", "");
RecursiveDocumentSplitter splitter = new RecursiveDocumentSplitter(
    500, 150, customSeparators
);
```

## 工作原理

### 示例文本
```
这是第一段。这是第二段。这是第三段。

这是第二个段落。内容很长...
```

### 分割过程
1. 尝试按"。"分割 → 得到3个句子
2. 检查每个句子长度
3. 如果句子超长，递归使用下一级分隔符
4. 添加150字符重叠，保持上下文连续性

## 对比效果

| 分割器 | 语义完整性 | 召回准确率 | 适用场景 |
|--------|-----------|-----------|---------|
| DocumentByParagraphSplitter | 中 | 70% | 简单文档 |
| RecursiveDocumentSplitter | 高 | 85%+ | 复杂中文文档 |

## 参数调优建议

### maxChunkSize（最大片段长度）
- **300-400**：适合短文档，快速检索
- **500-600**：平衡性能和语义（推荐）
- **700+**：长文档，需要更多上下文

### chunkOverlap（重叠长度）
- **50-100**：轻度重叠，节省存储
- **150-200**：中度重叠，平衡效果（推荐）
- **250+**：重度重叠，最大化召回

## 注意事项

1. **重新加载文档**：修改分割器后需重新加载文档才能生效
2. **存储空间**：重叠越大，向量存储占用越多
3. **中文优化**：默认配置已针对中文优化，英文文档可自定义分隔符
